1. 운영체제의 구성
- 메모리 관리: 메인 메모리 관리, 사용자의 프로세서 이용률과 컴퓨터 응답 속도 개선, 보조기억 장치 관리
- 프로세스 관리 : 일괄처리 작업, 시분할 사용자 프로그램, 스풀링 같은 시스템 작업 관리
- 장치 관리(입출력 관리) : 사용자가 특정 하드웨어 장치의 세부, 복잡성을 지닌 기계로 느끼지 못하도록 확장된 기계로 인식 시키는 것
- 파일 관리 : 물리적으로 여러가지 다른 형태의 정보 저장


2. 운영체제 서비스
- 부트스트래핑 서비스: 운영체제에서 자기자신을 적재하는 과정
- 사용자 서비스: 프로그래머가 작업을 쉽게 할 수 있도록 제공하는 서비스
- 시스템 서비스: 시스템 자체에 효율적인 동작을 보장하는 기능
- 시스템 호출 서비스: 실행 중인 프로그램과 운영체제 간의 인터페이스


3. 시스템 구조
- 간단한 구조: MS-DOS 운영체제, UNIX 운영체제
- 계층적 접근: 시스템의 모듈화, 운영체제를 다수의 계층들로 분할하여 구성, 각각 하위 계층의 위쪽에 배치
