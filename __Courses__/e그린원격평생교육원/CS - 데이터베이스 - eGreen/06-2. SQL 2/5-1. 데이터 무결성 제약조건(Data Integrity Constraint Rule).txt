=== 데이터 무결성 제약조건(Data Integrity Constraint Rule)

ㅇ 데이터 무결성 제약조건(Data Integrity Constraint Rule)
- 테이블에 부적절한 자료가 입력되는 것을 방지하기 위해서
- 테이블을 생성할 때 각 칼럼에 대해서 형식 하는 여러 가지 규칙

ㅇ 데이터 무결성
- 데이터베이스 내에 데이터의 정확성을 유지하는 것을 의미

ㅇ 데이터 무결성 제약조건 6가지
- NOT NULL: NULL 무결성 제약조건
- UNIQUE: 고유 무결성 제약조건
- PRIMARY KEY: 개체 무결성 제약조건
- FOREIGN KEY: 참조 무결성 제약조건
- CHECK: 도메인 무결성 제약조건
- 키 무결성 제약조건


ㅇ NOT NULL
- NULL 무결성 제약조건
- 특정 속성값에 NOT NULL 조건이 주어진 경우, 그 속성값은 NULL 값 불가능

ㅇ UNIQUE
- 고유 무결성 제약조건
- 특정 속성에 대해 고유한 값을 가지도록 조건이 주어진 경우, 그 속성값은 값 중복 불가능

ㅇ PRIMARY KEY
- 개체 무결성 제약조건
- NULL 값 가능, 값 중복 불가능
- NOT NULL + UNIQUE

ㅇ FOREIGN KEY
- 참조 무결성 제약조건
- 외래키 값은 NULL이거나 참조 릴레이션의 기본키 값과 동일해야 함

ㅇ CHECK
- 도메인 무결성 제약조건
- 특정 속성의 값이 그 속성이 정의된 도메인에 속한 값이어야 함

ㅇ 키 무결성 제약조건
- 한 릴레이션(테이블)에는 최소한 하나의 키가 존재해야 함
