

=== 엔티티, 관계 속성

ㅇ 데이터 모델링을 하기 위해서는 엔티티, 관계, 속성에 대한 이해가 필요
- ex) 이도령과 성춘향이 있고 둘은 사랑하는 사이다. 이도령은 키가 186이고 성격이 까칠하고, 성춘향은 키가 163이고 세심하고 활달한 성격이다.


ㅇ 개체(=엔티티 =ENTITY)
- 업무가 관여하는 어떤 것
- ex) "이도령", "성춘향"

- 데이터베이스에 자료로 표현하려는 것
- 사람이 생각하는 개념이나 정보단위 같은 현실 세계의 대상체
- 유형, 무형의 정보
- 서로 연관된 몇 개의 속성으로 구성
- ex) 부서 엔티티, 사원 엔티티


ㅇ 관계(RELATION)
- 업무가 관여하는 어떤 것 간의 관계
- ex) "사랑하는 사이"

- 엔티티와 엔티티 혹은 엔티티와 속성 간의 연관성을 관계라고 함
- 일반적으로 동사형으로 표현
- ex) 사원이 부서에 소속되어 있다.

ㅇ 속성(ATTRIBUTE)
- 어떤 것의 성격
- ex) "성춘향" : "163cm", "세심, 활달"

- 데이터의 가장 작은 논리적 단위
- 하나의 엔티티는 한 개 이상의 속성으로 구성
- 각 속성은 엔티티의 특성, 상태 등을 기술
- ex) 부서 엔티티의 속성: 부서 번호, 부서명, 위치
- ex) 사원 엔티티의 속성: 사원번호, 이름, 주소, 소속부서
